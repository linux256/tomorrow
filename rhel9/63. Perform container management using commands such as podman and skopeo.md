## MANAGE CONTAINERS
# 63 Perform container management using commands such as podman and skopeo

---

### I. EJECUCION DE CONTENEDORES

`podman run nginx`

> Ejecuta un contenedor generado desde la *container image* `nginx`. Al no
especificarlo, solicitará desde que *registry* descargar la imagen

`podman run docker.io/library/nginx`

> Descarga la imagen de `docker.io` y ejecuta el contenedor

**`podman run -d nginx`**

> Ejecuta el contenedor en segundo plano o modo *deattached*

**`podman run -it nginx`**

> Ejecuta el contenedor `nginx` y se conecta al *TTY*. No se podrá interactuar
si no ejecuta un shell.

**`podman run -it nginx /bin/sh`**

> Ejecuta el contenedor `nginx` y el comando `/bin/sh` en lugar de su comando
por defecto. Para abandonar el contenedor hay dos vías,

> * `exit` termina el comando y por tanto detiene el contenedor
> * **`CTRL-P , CTRL-Q`** deja el contenedor en ejecución en segundo plano

`podman attach  nombreContenedor`

> Se conecta al *TTY* del contenedor `nombreContenedor`

>

### II. LISTADO DE CONTENEDORES, PARADA Y BORRADO

**`podman ps`**

> Lista los contenedores en ejecución

**`podman ps -a`**

> Lista todos los contenedores generados, detenidos y en ejecución

`podman stop  nombreContenedor`

> Detiene el contenedor `nombreContenedor`

`podman rm  nombreContenedor`

> Borra el contenedor `nombreContenedor`

`~/.local/share/containers/storage/`

> Ruta donde se generan los contenedores *rootless*
