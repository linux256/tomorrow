## OPERATE RUNNING SYSTEMS
# 24. Start, stop and check the status of network services

---

### I. SYSTEMCTL

`dnf install bash-completion`

> Instala scripts que amplian las capacidades de bash para terminar comandos al
> tabular, empleando por ejemplo las opciones de muchos comandos

`systemctl start servicio`

> Intenta iniciar servicio

`systemctl status servicio`

> Muestra estado del servicio, que puede ser alguno de los siguientes valores
devueltos en dos líneas:

> **Loaded**

> * `loaded` el servicio (unit) está cargado en memoria
> * `error` error al cargar el servicio
> * `bad-setting` error el el unitfile de descripción del servicio
> * `masked` el servicio tiene prohibido iniciarse
> * `enabled` se inicia automáticamente en el arranque del sistema
> * `disabled` el servicio no se inicia en el arranque del sistema
> * `static` el servicio no está enabled pero lo puede iniciar automaticamente
> otro servicio

> **Active**

> * `active (running)` en ejecución
> * `active (exited)`
> * `active (waiting)`
> * `inactive (dead)`
> * `failed` el servicio ha fallado, *crash*,  o no responde, *timeout* 

`systemctl --state=help`

> Muestra todos los estados posibles de las unit

**`systemctl --state=running -t service`**

> Enumera todos los servicios en ejecución

**`systemctl --failed -t service`**

> Enumera todos los servicios fallados

**`systemctl --all -t service`**

> Enumera todos los servicios

**`systemctl -t service`**

> Enumera todos los servicios activos (mismo resultado que `systemctl
> --state=active -t service`)

**`systemctl list-units`**

> Lista las units cargadas en memoria y activas. `list-units` es el subcomando
> por defecto, puede omitirse (mismo resultado que `systemctl`)
