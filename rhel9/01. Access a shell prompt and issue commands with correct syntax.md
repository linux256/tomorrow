## UNDERSTAND AND USE ESSENTIAL TOOLS
# 01. Access a shell prompt and issue commands with correct syntax

---

### I. COMANDOS, ALIAS Y UBICACION. VARIABLES DE ENTORNO

`command options arguments`

> forma general de un comando en Linux

`alias`

> define un alias

`\comando`

> se ejecuta comando y no el alias "comando"

`time comando`

> muestra tiempo empleado en ejecutar comando

**`type comando`**

> permite saber si un comando es interno, externo, o un alias

**`which comando`**

> busca ruta del comando

**`whereis comando`**

> muestra ruta del comando, de sus paginas man y de su código fuente

`file fichero`

> muestra el tipo del fichero

`env`

> muestra variables del entorno; también las establece y ejecuta un comando

`$PATH`

> variable en la que se buscan los comandos para su ejecución (distinto en root)
> `~/.local/bin ~/bin  /usr/local/bin /usr/bin  /bin  /usr/local/sbin /usr/sbin`

### II. PERSONALIZACION Y SCRIPTS DE INICIO

`/etc/issue`

> mensaje mostrado *antes* del login

`/etc/motd`

> mensaje mostrado *despues* del login

`/etc/profile`

> script ejecutado, en todos los login shell

`~/.bash_profile`

> script ejecutado, en login shell del usuario

`/etc/bashrc`

> script ejecutado, en todos los subshells

`~/.bashrc`

> script ejecutado, en los subshells del usuario

### III. HISTORIAL DE COMANDOS

**`history`**

> muestra historial de comandos, accesible con `CTRL+r`, con `!num` o con `!str`
> para ejecutar el comando "num" o el último comando que comenzara por "str"

`~/.bash_history`

> history file

`history -a`

> añade historial de comandos de la sesión a history file

`history -c`

> borra historial de comandos de la sesión

`history -c && history -w`

> sobreescribe history filo con history de sesion

`history -d 27`

> borra comando 27 de la sesión
