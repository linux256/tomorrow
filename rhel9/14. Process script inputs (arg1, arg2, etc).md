# 14. Process script inputs ($1, $2, etc.)

### I. ARGUMENTOS

*Argumento: cualquier cosa que va detrás del comando del script al iniciarlo*

* `$0` argumento 0 del script o comando, es el propio nombre del script/comando
* `$1` primer argumento del script/comando
* `$2` segundo argumento del script/comando
* `...`
* `$9` noveno argumento

>

* `$#` número de argumentos del script/comando
* `$@` todos los argumentos indexados, en un array
* `$*` todos los argumentos en una cadena única

>

```
#!/bin/bash                             # shebang
echo -n $0                              # -n omite el salto de línea
for i in $@; do echo -n " $i"; done`    # la variable $i se interpreta entre ""
echo                                    # introducimos salto de línea final
```

> Muestra en pantalla el comando completo que se ha introducido. Mas sencillo:
> `echo $0 $*`

### II. VARIABLES

*Variable: etiqueta empleada para referirse a una posición de memoria que
contiene un valor/dato específico. Por convención se nombran en MAYÚSCULA*

`NAME="minombre"`

> Variable asignada estáticamente, **sin espacios ni antes ni despues de "="**

`FECHA=$(date "+%F %T")`

> Variable asignada dinámicamente

`read NAME`

> Variable asignada dinámicamente

`$VAR`

> Valor de la variable, contenido de la memoria apuntada por VAR

`echo "\$VAR es el contenido de VAR y vale $VAR"`

`echo '$VAR es el contenido de VAR y vale ' $VAR`

> El empleo de `'` permite emplear cadenas literales, sin interpretar `$VAR`.
> Otra opcion es escapar el simbolo `$` con `\$`

### III. EMPLEO DE `{}` EN ARGUMENTOS Y VARIABLES

`${10}`

> décimo argumento, se emplean `{}` para no confundirlo con `$1` y `0`

`${11}`

> undécimo argumento

`${1:-20}`

> primer argumento o el valor "20" si no está definido; no asigna valor a `$1`

`${1:-"--help"}`

> primer argumento o "help" si no se ha definido; no asigna valor a `$1`

`echo "$VAReur"`

> Fallará porque no existe la variable "VAReur"

`echo "${VAR}eur"`

> El empleo de `{...}` asegura que el nombre de la variable se interpreta bien

`echo ${A:-"no existe"}`

> Mostrará el valor de `A` o la cadena `no existe` si no está definido. Esto no
> conlleva asignar ningún valor a la variable `A`

`echo ${ARRAY[0]}`

> Muestra el primer elemento del array `ARRAY`
