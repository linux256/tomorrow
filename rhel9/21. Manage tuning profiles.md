## OPERATE RUNNING SYSTEMS
# 21. Manage tuning profiles

---

### I. DAEMON TUNED

`systemctl enable --now tuned`

> Habilita (crea un *wanted* para el target por defecto) el servicio `tuned` y
> lo inicia de inmediato. El servicio `tuned` gestiona perfiles de rendimiento,

> * `balanced` equilibrio entre consumo y rendimiento
> * `desktop` para responder a aplicacines interactivas
> * `powersave` para ahorra consumo eléctrico

> * `latency-performance` para optimizar rendimiento (ms)
> * **`throughput-performance`** para optimizar rendimiento de disco y red
> (Mb/s). Este perfil suele ser el perfil por defecto

> * `network-latency` para optimizar retraso de red
> * `network-throughput` para optimizar rendimiento de red

> * `virtual-host` cuando el sistema opera como anfitrión de máquinas virtuales
> * `virtual-guest` cuando el sistema opera como máquina virtual invitada

>

`tuned-adm list`

> Muestra perfiles disponibles

`tuned-adm active`

> Muestra perfil activo

`tuned-adm recommend`

> Muestra perfil recomendado

`tune-adm profile powersave`

> Activa el perfil de ahorro de consumo eléctrico
