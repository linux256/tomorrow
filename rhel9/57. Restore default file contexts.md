## MANAGE SECURITY
# 57. Restore default file contexts

---

### I. APLICACION DE FILE CONTEXT A NUEVOS FICHEROS

Los nuevos ficheros que se crean reciben un *file context* según las siguientes
normas,

1. El fichero hereda el contexto del directorio donde se encuentra

2. Si el fichero se copia a *destino*, hereda el contexto de *destino*

3. Si el fichero se mueve o se copia a *destino* con sus propiedades (`cp -a`),
mantiene el contexto original

> 

*Si un fichero se copia con `cp -aZ` se mantienen todas sus propiedades salvo el
contexto, que será por defecto*

>

### II. RESTABLECIMIENTO DE FILE CONTEXT

`restorecon -Rv /`

> Lanza ahora proceso de *relabel* completo del sistema de ficheros

`touch /.autorelabel`

> En el siguiente reinicio se realizará un *relabel* completo del filesystem
